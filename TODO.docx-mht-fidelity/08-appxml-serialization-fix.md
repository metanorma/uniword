# 08 - App.xml Serialization Fix ("Summary Info 1 Needs Repair")

## Status: DONE

## Problem

Any DOCX generated by `Uniword::Docx::Package.to_file` fails to open in Microsoft Word with the error:

> "Word experienced an error trying to open the file"
> Internal: "Summary Info 1 needs repair"

This affects ALL generated DOCX files, even minimal single-paragraph documents.

## Root Cause

`lutaml-model` tracks an internal `@using_default` hash that flags which attributes were set via their default values vs. explicitly assigned. During `to_xml` serialization, attributes still flagged as `using_default: true` are **silently omitted** from the XML output.

In `docx/reconciler.rb`, the `reconcile_app_properties` method uses `||=` for static defaults:

```ruby
app.total_time ||= "0"       # Never reassigns — value already equals default
app.doc_security ||= "0"     # @using_default stays true → omitted from XML
app.scale_crop ||= "false"   # Same
app.links_up_to_date ||= "false"
app.shared_doc ||= "false"
app.hyperlinks_changed ||= "false"
```

Since `AppProperties` defines these with matching defaults (`default: -> { "0" }`), the `||=` guard evaluates to "already set, skip assignment" — but lutaml-model still considers them "using default" and skips serialization.

**Result**: `docProps/app.xml` is missing `TotalTime`, `DocSecurity`, `ScaleCrop`, `LinksUpToDate`, `SharedDoc`, and `HyperlinksChanged`. Word flags this as corrupted "Summary Info".

## Fix

Changed `||=` to `=` in `reconcile_app_properties` to force explicit assignment and clear the `@using_default` flag:

```ruby
app.total_time = "0"
app.scale_crop = "false"
app.doc_security = "0"
app.links_up_to_date = "false"
app.shared_doc = "false"
app.hyperlinks_changed = "false"
```

File: `lib/uniword/docx/reconciler.rb`, `reconcile_app_properties` method.

## Related: lutaml-model `@using_default` Pattern

This is a general gotcha with lutaml-model. Any code that needs to ensure an attribute appears in serialized XML must use direct `=` assignment, not `||=`, even when the value matches the default. The `@using_default` hash controls serialization, and `||=` doesn't clear it.
