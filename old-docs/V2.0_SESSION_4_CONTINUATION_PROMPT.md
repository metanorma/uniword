## Implementation Strategy

### Step 1: Create DrawingML Schema (90 minutes)

Create [`config/ooxml/schemas/drawingml.yml`](config/ooxml/schemas/drawingml.yml):

```yaml
# DrawingML Main Namespace Schema
# ISO 29500-1 Part 4: DrawingML
# URI: http://schemas.openxmlformats.org/drawingml/2006/main
# Prefix: a

namespace:
  uri: 'http://schemas.openxmlformats.org/drawingml/2006/main'
  prefix: 'a'
  description: 'DrawingML - Graphics, shapes, and visual effects'

elements:
  # Define 150+ elements with:
  # - class_name: RubyClassName
  # - description: Clear purpose
  # - attributes: Array of attribute definitions
  #   - name, type, xml_name, xml_attribute, required, collection
```

**Use Lessons from Math Namespace**:
- Use `String` for boolean-like attributes
- Use `String` for undefined types
- Avoid cross-namespace dependencies initially
- Keep types simple (String, Integer, Float)

### Step 2: Generate Classes (10 minutes)

```ruby
require_relative 'lib/uniword/schema/model_generator'
generator = Uniword::Schema::ModelGenerator.new('drawingml')
results = generator.generate_all
puts "Generated #{results.size} DrawingML classes"
```

### Step 3: Create Autoload Index (15 minutes)

Create [`lib/generated/drawingml.rb`](lib/generated/drawingml.rb):

```ruby
module Uniword
  module Generated
    module DrawingML
      # Autoload all DrawingML classes (150+)
      autoload :Graphic, File.expand_path('drawingml/graphic', __dir__)
      autoload :GraphicData, File.expand_path('drawingml/graphic_data', __dir__)
      # ... 148 more autoload declarations
    end
  end
end
```

**Organization**:
1. Simple value classes first
2. Property classes second
3. Container classes third
4. Complex structures last

## Critical Principles

### 1. ZERO Raw XML Tolerance
No `UnknownElement`, no `raw_*` attributes, no exceptions.

### 2. Schema-Driven Everything
ALL structure in YAML, code generated automatically.

### 3. MECE Architecture
Each element: ONE responsibility, no overlap.

### 4. Object-Oriented Design
Models are pure Ruby objects, serialization is separate.

### 5. Pattern 0 Sacred
Attributes BEFORE xml blocks (enforced by generator).

### 6. Autoload Standard
All namespaces use autoload for efficient lazy loading.
