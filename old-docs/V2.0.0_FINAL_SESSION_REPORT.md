## Architectural Improvements

### 1. Clean Separation: Style ID vs. Style Name

**Before**:
```ruby
paragraph.style  # Returned "Heading1" (ID)
```

**After**:
```ruby
paragraph.style     # Returns "Heading 1" (Name) - user-facing API
paragraph.style_id  # Returns "Heading1" (ID) - programmatic API
```

**Benefits**:
- Clear API for different use cases
- User-facing methods return human-readable values
- Programmatic methods provide raw data for internal use

### 2. Round-Trip Fidelity Priority

**Principle**: Preserve all document elements including empty ones for accurate round-trip conversion

**Implementation**:
- Empty paragraphs preserved in HTML deserialization
- Whitespace-only content preserved
- All elements maintained regardless of perceived "emptiness"

**Benefits**:
- MHTML → Document → MHTML produces identical output
- No data loss during format conversion
- Better compatibility with Word behavior
