## Implementation Strategy

### Step 1: Create Math Schema (30 minutes)

Create [`config/ooxml/schemas/math.yml`](config/ooxml/schemas/math.yml):

```yaml
# MathML Namespace Schema
# ISO 29500-1 Part 1: Math Namespace
# URI: http://schemas.openxmlformats.org/officeDocument/2006/math
# Prefix: m

namespace:
  uri: 'http://schemas.openxmlformats.org/officeDocument/2006/math'
  prefix: 'm'
  description: 'Office Math Markup Language (OMML)'

elements:
  # Root elements
  oMath:
    class_name: OMath
    description: 'Math object container'
    attributes:
      - name: elements
        type: MathElement
        collection: true
        xml_name: '*'
        description: 'Math structure elements'
  
  # Add remaining 79+ elements...
```

### Step 2: Generate Classes (5 minutes)

```ruby
require_relative 'lib/uniword/schema/model_generator'
generator = Uniword::Schema::ModelGenerator.new('math')
results = generator.generate_all
puts "Generated #{results.size} Math classes"
```

### Step 3: Validate (5 minutes)

```bash
ruby -e "
require_relative 'config/ooxml/schemas/math.yml'
schema = YAML.load_file('config/ooxml/schemas/math.yml')
puts 'Elements: ' + schema['elements'].keys.size.to_s
"
```

### Step 4: Test (5 minutes)

```bash
ruby test_generator.rb
# Should show Math namespace with 80+ elements
```

## Schema Format Template

Use this template for each element (copy from wordprocessingml.yml):

```yaml
elements:
  elementName:
    class_name: RubyClassName
    description: 'Clear description of element purpose'
    attributes:
      - name: attribute_name
        type: AttributeType
        xml_name: xmlElementName
        xml_attribute: xmlAttrName  # For XML attributes
        required: true  # If mandatory
        collection: true  # If repeatable
        description: 'Attribute purpose'
```

## Pattern 0 Reminder

Generator enforces this automatically, but remember:

```ruby
# ✅ CORRECT
class Example < Lutaml::Model::Serializable
  attribute :my_attr, Type  # FIRST
  
  xml do
    root 'elem'
    namespace 'uri', 'prefix'
    map_element 'elem', to: :my_attr
  end
end

# ❌ WRONG
class Example < Lutaml::Model::Serializable
  xml do
    # ...
  end
  attribute :my_attr, Type  # TOO LATE!
end
```

## Critical Principles

### 1. ZERO Raw XML Tolerance
No `UnknownElement`, no `raw_*` attributes, no exceptions.

### 2. Schema-Driven Everything
ALL structure in YAML, code generated automatically.

### 3. MECE Architecture
Each element: ONE responsibility, no overlap.

### 4. Object-Oriented Design
Models are pure Ruby objects, serialization is separate.

### 5. Pattern 0 Sacred
Attributes BEFORE xml blocks (enforced by generator).
