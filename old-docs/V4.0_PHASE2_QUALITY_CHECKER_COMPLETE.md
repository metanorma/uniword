## Implementation Overview

### Architecture

The quality checking system follows SOLID principles and clean architecture:

```
Uniword::Quality
├── DocumentChecker      # Orchestrates quality checks
├── QualityReport        # Manages and exports results
├── QualityRule          # Base class for all rules
├── QualityViolation     # Represents individual violations
└── Rules/
    ├── HeadingHierarchyRule
    ├── TableHeaderRule
    ├── ParagraphLengthRule
    ├── ImageAltTextRule
    ├── LinkValidationRule
    └── StyleConsistencyRule
```

### Key Design Principles

1. **Single Responsibility**: Each rule checks ONE quality aspect
2. **Open/Closed**: New rules added via configuration without code changes
3. **Configuration over Convention**: All behavior defined in external YAML
4. **Separation of Concerns**: Checking ≠ Reporting ≠ Rules
5. **MECE Organization**: Each rule is mutually exclusive and collectively exhaustive

### 2. Quality Rules Implemented

#### [`HeadingHierarchyRule`](lib/uniword/quality/rules/heading_hierarchy_rule.rb)
**Purpose**: Ensures proper heading structure
**Checks**:
- Headings don't skip levels (H1 → H3)
- Heading levels don't exceed maximum (default: 6)
- Sequential hierarchy maintenance

**Configuration**:
```yaml
heading_hierarchy:
  enabled: true
  max_level: 6
  require_sequential: true
```

#### [`TableHeaderRule`](lib/uniword/quality/rules/table_header.rb)
**Purpose**: Validates table accessibility
**Checks**:
- Tables have header rows
- Headers are properly structured

**Configuration**:
```yaml
table_headers:
  enabled: true
  require_headers: true
```

#### [`ParagraphLengthRule`](lib/uniword/quality/rules/paragraph_length_rule.rb)
**Purpose**: Enforces readability standards
**Checks**:
- Paragraphs under maximum word count
- Warnings for long paragraphs
- Skips empty paragraphs

**Configuration**:
```yaml
paragraph_length:
  enabled: true
  max_words: 500        # Error threshold
  warning_words: 400    # Warning threshold
```

#### [`ImageAltTextRule`](lib/uniword/quality/rules/image_alt_text_rule.rb)
**Purpose**: Ensures document accessibility
**Checks**:
- Images have alt text
- Alt text meets minimum length

**Configuration**:
```yaml
image_alt_text:
  enabled: true
  require_alt_text: true
  min_length: 10
```

#### [`LinkValidationRule`](lib/uniword/quality/rules/link_validation_rule.rb)
**Purpose**: Validates hyperlink integrity
**Checks**:
- Internal bookmarks exist
- External URLs have valid format
- Optional network connectivity check

**Configuration**:
```yaml
link_validation:
  enabled: true
  check_internal: true
  check_external: false
```

#### [`StyleConsistencyRule`](lib/uniword/quality/rules/style_consistency_rule.rb)
**Purpose**: Enforces consistent styling
**Checks**:
- Paragraphs use standard styles
- Direct formatting detection
- Style compliance

**Configuration**:
```yaml
style_consistency:
  enabled: true
  allow_direct_formatting: false
  require_standard_styles: true
```

## Usage Examples

### Basic Usage

```ruby
require 'uniword'

# Load document
document = Uniword::Document.open('document.docx')

# Create checker with default rules
checker = Uniword::Quality::DocumentChecker.new

# Check document
report = checker.check(document)

# Check if valid
if report.valid?
  puts "✓ Document meets quality standards"
else
  puts "✗ #{report.error_count} errors found"

  # Display violations
  report.violations.each do |v|
    puts "  [#{v.severity}] #{v.message} (#{v.location})"
  end
end
```

### Custom Configuration

```ruby
# Use custom rules file
checker = Uniword::Quality::DocumentChecker.new(
  rules_file: 'custom_quality_rules.yml'
)

# Or provide configuration directly
config = {
  quality_rules: {
    paragraph_length: {
      enabled: true,
      max_words: 300,
      warning_words: 250
    }
  }
}

checker = Uniword::Quality::DocumentChecker.new(config: config)
```

### Report Export

```ruby
report = checker.check(document)

# Export as JSON
report.to_json('quality_report.json')

# Export as YAML
report.to_yaml('quality_report.yml')

# Export as HTML
report.export_html('quality_report.html')

# Get text summary
puts report.summary
```

### Programmatic Access

```ruby
report = checker.check(document)

# Get violations by severity
errors = report.by_severity[:error]
warnings = report.by_severity[:warning]
info = report.by_severity[:info]

# Get violations by rule
heading_issues = report.by_rule['heading_hierarchy']

# Get statistics
puts "Total: #{report.total_count}"
puts "Errors: #{report.error_count}"
puts "Warnings: #{report.warning_count}"
puts "Info: #{report.info_count}"

# Check enabled rules
puts "Enabled: #{checker.enabled_rules.join(', ')}"
puts "Disabled: #{checker.disabled_rules.join(', ')}"
```
