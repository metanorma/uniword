## Feature 1: Link Validator

### What It Is

**Purpose**: Validate ALL links and references in documents to ensure they're not broken.

**User Problem**:
- Documents have hundreds of hyperlinks to external websites, internal bookmarks, or other documents
- Over time, websites go down, bookmarks get renamed, files get moved
- Publishing documents with broken links damages credibility
- Manual link checking is impossible at scale

**What It Does**:
- Scans document for ALL links (hyperlinks, cross-references, footnotes, bookmarks)
- Validates each link:
  - **External URLs**: HTTP requests to check if website exists
  - **Internal bookmarks**: Verify bookmark exists in document
  - **File references**: Check if referenced file exists
  - **Footnote/Endnote references**: Verify targets exist
- Generates detailed report of broken links
- Optional: Auto-fix common issues (e.g., update URLs)

**Example Use Case**:
```
ISO Standard Document has:
- 50 external links to specifications
- 200 internal cross-references to sections
- 100 footnote citations
- 30 bookmark links

Link Validator:
1. Checks all 380 references
2. Finds 5 broken external links (websites down)
3. Finds 2 missing bookmarks
4. Generates report with locations and suggestions
5. Author fixes before publication
```

### Architecture Design

```ruby
module Uniword
  module Validation
    # Link Validator - validates all document references
    #
    # Responsibility: Validate links and cross-references
    # Single Responsibility: Link validation only
    #
    # External config: config/link_validation_rules.yml
    #
    # Architecture:
    # - LinkValidator: Orchestrator
    # - LinkChecker: Base class for checkers
    # - ExternalLinkChecker: HTTP validation
    # - InternalLinkChecker: Bookmark validation
    # - FileReferenceChecker: File existence
    # - ValidationReport: Results
    class LinkValidator
      def initialize(config_file: 'config/link_validation_rules.yml')
        @config = load_config(config_file)
        @checkers = initialize_checkers
      end

      # Validate all links in document
      # Returns ValidationReport with all issues
      def validate(document)
        report = ValidationReport.new

        # Check hyperlinks
        document.hyperlinks.each do |link|
          result = check_link(link)
          report.add_result(link, result) unless result.valid?
        end

        # Check bookmarks
        validate_bookmarks(document, report)

        # Check footnote references
        validate_footnotes(document, report)

        report
      end

      private

      # Check single link using appropriate checker
      def check_link(link)
        checker = @checkers.find { |c| c.can_check?(link) }
        checker ? checker.check(link) : ValidationResult.unknown(link)
      end
    end

    # External URL checker
    class ExternalLinkChecker < LinkChecker
      def can_check?(link)
        link.url && link.url.match?(/^https?:\/\//)
      end

      def check(link)
        response = HTTP.get(link.url, timeout: @config[:timeout])
        ValidationResult.success(link)
      rescue HTTP::Error
        ValidationResult.failure(link, "URL not accessible: #{link.url}")
      end
    end

    # Bookmark checker
    class InternalLinkChecker < LinkChecker
      def can_check?(link)
        link.anchor && !link.url
      end

      def check(link)
        if document.bookmarks.key?(link.anchor)
          ValidationResult.success(link)
        else
          ValidationResult.failure(link, "Bookmark not found: #{link.anchor}")
        end
      end
    end
  end
end
```

## Feature 2: Metadata Manager

### What It Is

**Purpose**: Extract, manage, and update document metadata for indexing, search, and organization.

**User Problem**:
- Organizations have thousands of documents
- Need to catalog/index documents (title, author, date, keywords, subject)
- Need to search documents by metadata
- Need to update metadata in bulk (e.g., change author across 100 docs)
- Manual metadata entry is error-prone

**What It Does**:
- Extracts ALL metadata from documents:
  - Core properties (title, author, subject, keywords, comments)
  - Extended properties (company, manager, category)
  - Custom properties (user-defined key-value pairs)
  - Statistical metadata (word count, page count, paragraph count)
  - Content metadata (headings, images, tables)
- Updates metadata in documents
- Exports metadata to database/search index (JSON, CSV, database)
- Validates metadata against schema
- Bulk updates across document sets

**Example Use Case**:
```
Document Library Migration:
- 500 old documents need metadata
- Extract metadata from all 500 documents
- Export to CSV for review
- Update author field in bulk (Old Corp → New Corp)
- Re-import corrected metadata
- Index in search system (Elasticsearch, etc.)
```

### Architecture Design

```ruby
module Uniword
  module Metadata
    # Metadata Manager - extract and manage document metadata
    #
    # Responsibility: Metadata extraction and management
    # Single Responsibility: Metadata operations only
    #
    # External config: config/metadata_schema.yml
    #
    # Architecture:
    # - MetadataManager: Orchestrator
    # - MetadataExtractor: Extract from document
    # - MetadataUpdater: Update document properties
    # - MetadataValidator: Validate against schema
    # - MetadataIndex: Batch operations
    class MetadataManager
      def initialize(schema_file: 'config/metadata_schema.yml')
        @schema = load_schema(schema_file)
        @extractor = MetadataExtractor.new(@schema)
        @updater = MetadataUpdater.new(@schema)
        @validator = MetadataValidator.new(@schema)
      end

      # Extract all metadata from document
      def extract(document)
        metadata = @extractor.extract_all(document)
        @validator.validate(metadata)
        metadata
      end

      # Update document metadata
      def update(document, metadata)
        @validator.validate(metadata)
        @updater.apply(document, metadata)
      end

      # Batch extract from multiple documents
      def extract_batch(document_paths)
        index = MetadataIndex.new

        document_paths.each do |path|
          doc = Document.open(path)
          metadata = extract(doc)
          metadata[:file_path] = path
          index.add(path, metadata)
        end

        index
      end
    end

    # Metadata extractor
    class MetadataExtractor
      def extract_all(document)
        {
          # Core properties
          title: extract_title(document),
          author: extract_author(document),
          subject: extract_subject(document),
          keywords: extract_keywords(document),
          comments: extract_comments(document),

          # Extended properties
          company: extract_company(document),
          manager: extract_manager(document),
          category: extract_category(document),

          # Statistical
          word_count: document.text.split.count,
          paragraph_count: document.paragraphs.count,
          table_count: document.tables.count,
          image_count: document.images.count,
          page_count: estimate_page_count(document),

          # Content metadata
          headings: extract_headings(document),
          first_paragraph: document.paragraphs.first&.text,

          # Timestamps
          created: extract_created_date(document),
          modified: extract_modified_date(document),
          last_saved_by: extract_last_saved_by(document)
        }
      end

      private

      def extract_title(document)
        # From document properties, or first heading, or filename
        document.title ||
          document.paragraphs.find { |p| p.style == 'Title' }&.text ||
          'Untitled Document'
      end

      def extract_headings(document)
        document.paragraphs
          .select { |p| p.style&.start_with?('Heading') }
          .map { |p| { level: heading_level(p), text: p.text } }
      end
    end

    # Metadata index for batch operations
    class MetadataIndex
      def initialize
        @entries = []
      end

      def add(path, metadata)
        @entries << { path: path, metadata: metadata }
      end

      # Export to JSON for search indexing
      def to_json(file_path)
        File.write(file_path, JSON.pretty_generate(@entries))
      end

      # Export to CSV for spreadsheet
      def to_csv(file_path)
        CSV.open(file_path, 'w') do |csv|
          csv << metadata_keys
          @entries.each do |entry|
            csv << metadata_values(entry[:metadata])
          end
        end
      end

      # Bulk update
      def update_field(field, old_value, new_value)
        @entries.select { |e| e[:metadata][field] == old_value }
                .each { |e| e[:metadata][field] = new_value }
      end
    end
  end
end
```

## Feature 3: Document Assembler

### What It Is

**Purpose**: Build large documents from reusable components/sections.

**User Problem**:
- Large documents (100+ pages) are hard to maintain as single files
- Multiple authors need to work on different sections
- Common sections (legal disclaimers, standard text) are duplicated across documents
- Updates to shared content require editing many documents
- Version control of large files is problematic

**What It Does**:
- Stores document sections as separate files (components)
- Assembles final document from manifest/recipe
- Supports variables/placeholders in components
- Handles conditional inclusion of sections
- Manages cross-references between components
- Updates table of contents automatically

**Example Use Case**:
```
ISO Standard Document Assembly:

Components:
- cover_page.docx (template with variables: {title}, {date}, {version})
- legal_notice.docx (standard text, reused across all standards)
- scope.docx (specific to this standard)
- references.docx (specific to this standard)
- terms.docx (specific to this standard)
- clauses/*.docx (10 clause files)
- annexes/*.docx (5 annex files)

Manifest (assembly.yml):
```yaml
document:
  template: iso_standard_template
  output: iso-8601-2-2026.docx

  variables:
    title: "ISO 8601-2:2026"
    date: "2026-01-15"
    version: "Working Draft"

  sections:
    - component: cover_page
    - component: legal_notice
    - component: toc  # Auto-generated
    - component: scope
    - component: references
    - component: terms
    - component: clauses/*  # All files in directory
      order: alphabetical
    - component: annexes/*
      order: alphabetical
```

Result:
- Single command assembles final 100+ page document
- Updates to legal_notice.docx auto-propagate to all documents
- TOC generated automatically
- Variables substituted
- Cross-references updated
```

### Architecture Design

```ruby
module Uniword
  module Assembly
    # Document Assembler - build documents from components
    #
    # Responsibility: Assemble documents from reusable components
    # Single Responsibility: Document assembly only
    #
    # External config: assembly manifest (YAML)
    #
    # Architecture:
    # - DocumentAssembler: Orchestrator
    # - ComponentRegistry: Manage component library
    # - VariableSubstitutor: Replace placeholders
    # - TocGenerator: Auto-generate TOC
    # - CrossReferenceResolver: Update cross-refs
    class DocumentAssembler
      def initialize(components_dir: 'components/')
        @components = ComponentRegistry.new(components_dir)
      end

      # Assemble document from manifest
      def assemble(manifest_file)
        manifest = load_manifest(manifest_file)
        doc = create_base_document(manifest)

        # Apply template if specified
        apply_template(doc, manifest) if manifest['template']

        # Add sections in order
        manifest['sections'].each do |section|
          add_section(doc, section, manifest['variables'])
        end

        # Post-processing
        update_toc(doc) if manifest['generate_toc']
        resolve_cross_references(doc)

        doc
      end

      private

      def add_section(doc, section_config, variables)
        component_path = section_config['component']

        # Handle wildcards (e.g., clauses/*)
        if component_path.include?('*')
          add_wildcard_sections(doc, component_path, section_config, variables)
        else
          add_single_section(doc, component_path, variables)
        end
      end

      def add_single_section(doc, component_path, variables)
        component = @components.load(component_path)

        # Substitute variables
        if variables
          substitutor = VariableSubstitutor.new(variables)
          component = substitutor.apply(component)
        end

        # Add to document
        component.elements.each { |e| doc.add_element(e) }
      end
    end

    # Component registry
    class ComponentRegistry
      def initialize(base_dir)
        @base_dir = base_dir
        @cache = {}
      end

      def load(component_path)
        full_path = File.join(@base_dir, "#{component_path}.docx")

        # Cache components
        @cache[component_path] ||= Document.open(full_path)
      end

      def list(pattern: nil)
        Dir.glob(File.join(@base_dir, pattern || '**/*.docx'))
      end
    end

    # Variable substitutor
    class VariableSubstitutor
      def initialize(variables)
        @variables = variables
      end

      # Replace all {variable} placeholders in document
      def apply(document)
        document.paragraphs.each do |para|
          para.runs.each do |run|
            @variables.each do |key, value|
              run.text = run.text.gsub("{#{key}}", value.to_s)
            end
          end
        end
        document
      end
    end

    # TOC generator
    class TocGenerator
      def generate(document, max_level: 3)
        toc_doc = Document.new

        # Extract headings
        headings = document.paragraphs.select do |p|
          p.style&.match(/^Heading (\d+)$/) && $1.to_i <= max_level
        end

        # Create TOC entries
        headings.each do |heading|
          level = heading.style.match(/\d+/)[0].to_i

          toc_para = Paragraph.new
          toc_para.add_text("  " * (level - 1) + heading.text)
          toc_para.set_style('TOC' + level.to_s)

          toc_doc.add_paragraph(toc_para)
        end

        toc_doc
      end
    end
  end
end
```

### Configuration (assembly.yml example)

```yaml
# Document assembly manifest
document:
  template: iso_standard_template
  output: iso-8601-2-2026.docx

  variables:
    title: "ISO 8601-2:2026"
    date: "2026-01-15"
    version: "Working Draft"
    status: "Draft International Standard"

  sections:
    # Cover page with variables
    - component: templates/cover_page
      variables:
        title: "{title}"
        date: "{date}"

    # Standard legal notice (shared component)
    - component: shared/legal_notice

    # Auto-generated TOC
    - component: __toc__  # Special: auto-generate
      options:
        max_level: 3
        include_page_numbers: true

    # Content sections
    - component: sections/scope
    - component: sections/normative_references
    - component: sections/terms_and_definitions

    # All clauses (wildcard)
    - component: clauses/*
      order: alphabetical

    # All annexes
    - component: annexes/*
      order: alphabetical

  post_processing:
    generate_toc: true
    resolve_cross_references: true
    validate_quality: true
    quality_rules: config/quality_rules.yml
```

### Usage Example

```ruby
# Assemble document from manifest
assembler = Uniword::Assembly::DocumentAssembler.new(
  components_dir: 'document_components/'
)

doc = assembler.assemble('assembly.yml')

# Document is now fully assembled with:
# - Cover page with variables substituted
# - Legal notice included
# - Auto-generated TOC
# - All sections in correct order
# - Cross-references resolved

doc.save('assembled_document.docx')

# Update manifest and reassemble - changes propagate automatically
# Update shared/legal_notice.docx - applies to all documents
```

## Architectural Principles for All Features

**1. External Configuration**:
- All behavior defined in YAML
- No hardcoding of rules/logic
- Easy customization

**2. MECE Organization**:
- Each module handles ONE feature
- No overlapping responsibilities
- Complete coverage of use cases

**3. Separation of Concerns**:
- Validation ≠ Reporting
- Extraction ≠ Updating
- Assembly ≠ Components

**4. Open/Closed**:
- Add link checkers via configuration
- Add metadata fields via schema
- Add processing stages via manifest

**5. Single Responsibility**:
- LinkValidator: Validates links only
- MetadataManager: Manages metadata only
- DocumentAssembler: Assembles documents only
