## Critical Decision: ZERO RAW XML Tolerance

**Principle**: NO RAW XML STORAGE - EVER. Complete lutaml-model coverage or nothing.

**Rationale**: If we accept raw XML for some elements, we might as well accept raw XML for ALL elements. The entire point is COMPLETE OOXML MODELING using lutaml-model classes.

### Phase 1.2: Schema System Infrastructure ✅

**Created**:
- `config/ooxml/schemas/wordprocessingml.yml` - YAML schema for w: namespace (7 elements)
- `lib/uniword/schema/schema_loader.rb` - Schema loading and caching system
- `lib/uniword/schema/model_generator.rb` - YAML-to-lutaml-model class generator

**Schema Format**:
```yaml
namespace:
  uri: 'http://schemas.openxmlformats.org/wordprocessingml/2006/main'
  prefix: 'w'

elements:
  p:
    class_name: Paragraph
    description: 'Paragraph - block-level text element'
    attributes:
      - name: properties
        type: ParagraphProperties
        xml_name: pPr
      - name: runs
        type: Run
        collection: true
        xml_name: r
```

### Phase 1.3: Model Generator ✅

**Critical Feature**: Enforces Pattern 0 (attributes BEFORE xml blocks)

**Generated Class Example**:
```ruby
class Paragraph < Lutaml::Model::Serializable
  # ✅ ATTRIBUTES FIRST (Pattern 0)
  attribute :properties, ParagraphProperties
  attribute :runs, Run, collection: true, default: -> { [] }

  # ✅ XML MAPPING SECOND
  xml do
    root 'p'
    namespace 'http://...', 'w'
    mixed_content
    map_element 'pPr', to: :properties
    map_element 'r', to: :runs
  end
end
```

## Architecture Principles (Reinforced)

### The Most Critical Rule (Pattern 0)

**Attributes MUST be declared BEFORE xml mappings in lutaml-model classes**

This was the root cause of v1.1.0's serialization failures. Our generator now enforces this automatically.

### No Raw XML - Ever

UnknownElement was an anti-pattern that defeated the entire purpose. If we accept raw XML anywhere, we might as well accept it everywhere.

### Schema-Driven Everything

No hardcoding. Everything defined in YAML, generated automatically. This is the ONLY way to achieve 100% OOXML coverage.
